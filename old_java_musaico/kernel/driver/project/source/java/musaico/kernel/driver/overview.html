The library driver interfaces for Musaico.

Drivers provide access to persistent storage (SQL databases, file systems), networking (sockets, HTTP, and so on), in-memory information (reflection/introspection and so on), and any other object and document access points desired.

Each Driver provides read/write functionality as well as more specialized request commands.

Drivers can be stacked on top of other Drivers to provide "Driver stacks".

For example, a filter to read/write XML data might be stacked on top of a SQL database Driver or an HTTP client Driver to parse and format XML messages for persistent storage (for example, for guaranteed delivery) or to send to a web server.

The library drivers are modeled loosely after Linux device drivers. An excellent treatise on the subject:

Linux Device Drivers (current edition at the time of writing)
Linux Device Drivers (older edition)

The concept of stackable Drivers is modeled very loosely on the SVR3 STREAMS framework, though it is also meant to be much simpler.
