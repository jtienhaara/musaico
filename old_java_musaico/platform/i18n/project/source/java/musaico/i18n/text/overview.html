Simple internationalized text, dates, numbers, currency and so on.

For example, to create an internationalized piece of text so that it can be displayed localized to any user: 

      I18nString my_i18n_text = new I18nString ( "Hello, world." );

The localized form of this text can then be retrieved with: 

      Locale user_locale = ...; // For example, Locale.getDefault ().
      String display_to_user =
          my_i18n_text.localize ( user_locale )
          .value ();

Depending on the user's locale, the international string "Hello, world." would then be displayed to the user as (for example) "Bonjour, le monde."

Various MessageFormat s can be used to format internationalized messages with parameters, but the default provided with Musaico treats [%xyz%] blocks as parameter placeholders. For example, the following code will create an international message to say with parameter name = "Johann", and then localize it to "Bonjour, Johann": 

      I18nString my_i18n_text = new I18nString ( "Hello, [%name%]",
                                                 "name", "Johann" );
      String my_localized_text =
          my_i18n_text.localize ( Locale.FRENCH ).value ();

Each internationalized object must be accompanied with localized Messages file(s) in the package of the class which creates the internationalized object. For example, the following class: 

      package com.example.tryi18n;

      public class TryI18n
      {
          ...
          public I18nString hello ( String first_name,
                                    String last_name )
          {
              return new I18nString ( "Hello, [%first_name%] [%last_name%]",
                                      "first_name", first_name,
                                      "last_name", last_name );
          }
      }

Would require a Java resource bundle called com.example.tryi18n.Messages.properties to be loadable. The message bundle might look like: 

      Hello,\ [%first_name%]\ [%last_name%] = Hello, [%first_name%] [%last_name%]

This would provide the default (in this case English) localized message.

A French message bundle, com.example.tryi18n.Messages_fr.properties , might look like: 

      Hello,\ [%first_name%]\ [%last_name%] = Bonjour, [%first_name%] [%last_name%]

A Korean message bundle, com.example.tryi18n.Messages_kr.properties , might look something like: 

      Ahn nyeong [%last_name%] [%first_name%]

And so on.
