Atomic components of records and objects in Musaico.

Akin to the byte in an operating system, Musaico does not provide facilities for anything "smaller" than a Field.

A Field is comprised of an identifier (the field's "name") and a value.

The identifier for a Field is always an arbitrary String. It need not be unique in any scope whatsoever (although particular record types may restrict this freedom).

The value must be a "Musaico primitive", such as integer, String, date, time, reference, and so on.

See the individual Field implementations for details.

Musaico can easily be extended to incorporate other "primitive" Field values. See the (as yet unwritten) section on primtive typecast inferences.
